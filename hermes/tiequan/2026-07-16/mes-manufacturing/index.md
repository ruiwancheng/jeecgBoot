## 核心结论

**模块当前不可上线。** picking/completion 两模块表名与建表 SQL 不一致，部署即报表不存在；完工入库无累计校验可无限超计划、领料不与 BOM 对账、库存台账不联动，资损防线为零；订单无状态机永远草稿、completedQty 死字段，业务闭环断裂。

**共识 P0 共 7 项**（必改）：
1. picking/completion 表名不一致（实体 mes_* vs SQL c_mes_*）
2. 完工入库无累计校验（与 e84c96d 同型漏洞）
3. 领料不与 BOM 对账，可领任意物料任意数量
4. 库存台账断裂，领料/入库均不写台账
5. 订单无状态机 + completedQty 死字段 + 领料/入库状态校验方向反了
6. 菜单权限 sys_permission 未注册，接口全 403
7. 删除/编辑不校验下游引用，产生悬空外键/孤儿单

**修复顺序**：先修表名+菜单（能跑）→ 再修累计校验+BOM对账（防资损）→ 最后补状态机+台账+引用校验（业务闭环）。
