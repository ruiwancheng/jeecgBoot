[2026-07-08] [客户端开发] 客户端-服务端分离架构下的开发工作流
触发条件：在客户端-服务端分离架构（客户端只跑前端 pnpm dev，服务端运行后端和数据库）下开发
处理方式：
1. 前端改 Vue/TS：保存即生效，无需部署（Vite HMR + API 代理到服务端）
2. 新增前端页面：创建 .vue 文件 + 更新路由后，本地 dev 直接能访问（通过 URL），但菜单不显示直到服务端注册
3. 改后端 Java：需要 push → 部署控制台触发部署 → 服务端重启生效
4. 新增菜单/字典/表结构：需要 push + 部署，表结构通过 deploy.sh 第7步自动执行 SQL
5. 本地 dev 模式和服务器生产模式用同一个后端 API，但菜单数据来自后端数据库，本地需登出再登入刷新菜单缓存
6. 服务端部署后，前端 dev server 有时需要重启才能识别新的路由文件（Vite import.meta.glob 缓存）
