# PRD 核心逻辑和操作演示必须一起读——单看一个会漏数据结构

## 触发条件
开发销售出库模块时，只读了 PRD 核心逻辑（"出库单由发货单触发"），判断为引用模式单表结构。后来补读操作演示才发现"选择来源发货单，系统自动带出产品、数量"——明确指向主子表。

## 根因
核心逻辑描述概念（"出库单关联发货单"），操作演示描述交互（"带出产品、数量、输入实际出库数"）。交互隐含了数据模型——如果能输入数量，必然有明细行。

## 正确做法
读 PRD 时：核心逻辑 → 确定业务规则 → 操作演示 → 推导字段和数据结构。操作演示里的动词决定表结构（"带出"=关联查询，"输入"=需要独立字段，"多行"=主子表）。
