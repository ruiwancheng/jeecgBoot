# [2026-07-21] [安全] 给接口补权限注解前，先评估调用方范围

## 触发条件
审计发现"接口缺权限注解"，准备直接补上时。

## 现象
给取号接口 nextCode 补 `@RequiresPermissions("mes:codeRule:list")` 看似标准修复，
但调用方是 10 个单据页面的全体开单用户（销售/采购/生产/财务），
他们大多没有"编码规则"管理权限 → 自动编号对他们**静默失效**（前端 try/catch 回退手工输入），修复变倒退。

## 正确处理
1. 先列出接口的全部调用方（前端页面/其他服务），确认他们的权限覆盖面
2. 调用方权限不统一 → 不加注解，改为**注释说明放行理由**（谁、为什么、风险是否可接受）
3. 平台已强制登录的查询/取号类只读接口，无越权写入风险时可接受放行

## 检查清单
"缺权限注解"的审计项 ≠ 必须加注解。先问：谁在调？他们有这个权限吗？加了会不会误伤？
