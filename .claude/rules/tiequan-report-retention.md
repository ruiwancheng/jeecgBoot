# 铁拳团审计报告留存规则

## 规则

1. **按模块只保留最新一份审计报告。**
2. **旧模块报告直接删除，不做备份归档。**
3. **新报告生成后，必须删除同一模块的历史报告目录。**

## 目录结构

```
hermes/tiequan/YYYY-MM-DD/<模块名>/
```

## 报告组成

每个模块的报告包含：

| 文件 | 说明 |
|---|---|
| `01_风控总报告.md` | 汇总审计结果与修复建议 |
| `02_成员产出统计.md` | 审计团队产出统计 |
| `03-12_各成员专项报告.md` | 10 人铁拳团成员专项报告 |
| `index.md` | 报告索引与结论 |
| `13_复评总报告.md` | 复评时生成，如有 |

## 清理时机

- 同一模块再次审计时
- 产品决策调整导致复评报告替代原报告时
- 其他模块新报告生成后，不影响历史模块，但同一模块历史必须删除

## 禁止事项

- 禁止将报告放到 `hermes/audits/` 或其他目录
- 禁止保留同一模块的多份历史报告
- 禁止手动归档旧报告

## 示例

若已有 `hermes/tiequan/2026-07-09/mes-customer/`，后续又对 `mes-customer` 模块审计，生成 `hermes/tiequan/2026-07-13/mes-customer/` 时，必须删除 `2026-07-09/mes-customer/`。

不同模块（如 `mes-customer` 和 `warehouse-zone-shelf-location`）可分别保留各自最新一份。
