# /verify — 变更点自验证方案

## 问题

当前工作流缺少 AI 自验证环节。

```
修改代码 → 回归测试(/test-api /test-e2e) → 人工验收
              ↑
          测已有功能，不测本次变更点
```

## 设计：`/verify [客户名] [功能名]`

根据 `git diff` 自动分析变更类型，推导验证步骤并执行。放在写代码和回归测试之间。

### 变更类型 → 验证推导

| 变更类型 | git diff 特征 | 验证方式 |
|---------|-------------|---------|
| 新增接口 | Controller 新增方法 | curl 调用检查返回 |
| 修改接口 | Controller 代码变更 | curl 调用对比结果 |
| 新增页面 | 新增 .vue 文件 | Playwright 打开页面检查元素 |
| 修改组件 | .vue 变更 | 检查变更的元素是否存在/可用 |
| 修改实体 | Entity 字段变更 | API 返回检查字段 |
| 修改配置 | .yml .env 变更 | 检查配置项生效 |
| 新增 SQL | sql/ 新增 | 检查表已创建 |

### 示例

```
/verify demo 仓库管理
  git diff: WarehouseDrawer.vue 变更
  类型: 修改组件
  推导: 点新增 → 检查抽屉 → 确认/取消按钮存在
  执行: ✓ 确认按钮存在 ✓ 取消按钮存在
```

### 工作流位置

```
/brainstorm → /plan → 写代码 → /verify → 回归测试 → /done
                               │
                          改了什么验什么
                          (新增/修改/修复都适用)
```

## 落地清单

1. 创建 `/verify` 命令
2. 更新 `workflow.md` 加自验证环节
3. 更新 `/done` 检查清单
4. `/debug` 修复后自动提示 `/verify`
