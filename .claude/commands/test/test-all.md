# /test-all <项目名> [功能名]

API + E2E 全量测试。自动修复 + 汇总报告。

## 用法
- `/test-all demo` — demo 全部功能
- `/test-all demo 仓库管理` — 只测仓库管理

## 流程

### 1. 加载领域知识
使用 `test-all` 技能获取：测试聚合逻辑、智能范围判断参数、目录结构、汇总报告格式。

### 2. 智能范围判断（可选增强）
使用技能中的智能范围判断参数调用 MCP graph tools。
- 如果图形可用：低风险+少量变更 → 轻量模式（仅测受影响模块）；高风险 → 全量模式
- 如果图形不可用：全量模式

### 3. 扫描
检查已有测试，缺则自动生成。

### 4. API 测试
逐模块运行，失败则自动修复后重跑。

### 5. E2E 测试
浏览器全流程，失败则自动修复后重跑。

### 6. 汇总报告
按技能中的汇总报告格式输出。
