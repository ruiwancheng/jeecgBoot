# /test-api <项目名> <功能名>

自动运行接口测试。发现问题后自动尝试修复，不需要人工介入。

## 流程

### 1. 加载领域知识
使用 `test-api` 技能获取：错误诊断与修复映射表、JeecgBoot API 响应格式规范、认证配置信息、智能测试范围参数。

### 2. 智能范围判断（可选增强）
使用技能中的智能测试范围参数调用 MCP graph tools。
- 如果图形可用：精准确定变更影响哪些 Controller，只跑相关测试
- 如果图形不可用：全量跑

### 3. 找测试
检查 `harness/tests/<项目名>/<功能名>.test.js`，没有则自动生成。

### 4. 跑测试
运行测试文件。

### 5. 通过 → 直接告知
```
<功能名> API 测试：全部通过。
```

### 6. 失败 → 自动修复 → 重测
按技能中的错误诊断映射表匹配报错类型，执行对应修复方案。修复后重跑，最多尝试 3 次。修复失败则告知具体报错。
