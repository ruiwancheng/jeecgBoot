# /test-frontend <项目名> <功能名>

前端快速自测，不打开浏览器，秒级出结果。适合日常开发中频繁使用。

## 流程

### 1. 加载领域知识
使用 `test-frontend` 技能获取：组件完整性检查模式、路由检查规范、TypeScript 类型检查命令、构建验证命令、组件引用验证增强参数。

### 2. 组件引用验证（可选增强）
使用技能中的组件引用验证参数调用 MCP graph tools。
- 如果图形可用：通过图谱验证 import 路径和路由 component 指向都真实存在
- 如果图形不可用：标准文件系统检查

### 3. 组件完整性
检查页面文件引用的子组件都存在。

### 4. 路由检查
验证路由文件指向的组件是否存在。

### 5. 类型检查
运行 TypeScript 类型检查。

### 6. 构建验证
运行生产构建。

### 7. 输出
每项通过/失败，失败时列出具体文件和原因。
