# /debug <报错描述>

**铁律：未找到根因不能修。同一修复 3 次仍失败 → 停止，质疑架构。**

## 流程

### 1. 加载领域知识
使用 `debug` 技能获取：常见报错速查、3次即停判定标准、gen-tests-rules.json 字段 schema、Bug 反哺决策矩阵。

### 2. 诊断
读报错 → 定位文件+行号 → 分析类型 → 检查 `git diff` 近期改动。

### 3. 最小修复
一次只改一个变量。

### 4. 验证
运行 `/verify`。

### 5. 3次即停
同一问题修 3 次还失败 → 告知用户疑是架构问题。

### 6. 修复完成后的反哺
分析 Bug 根因，判断 gen-tests 是否漏了此类场景。如能通过规则改进覆盖，询问用户是否加入推导规则。不能覆盖的（架构/环境问题）记录到 learnings 即可。
