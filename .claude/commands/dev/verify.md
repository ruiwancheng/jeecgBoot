# /verify [项目名] [功能名]

**铁律：没有具体证据，不能声称"通过了"。**

## 流程

### 1. 加载领域知识
使用 `verify` 技能获取：变更类型到验证方法的映射表。

### 2. 图形预分析（可选增强）
使用技能中的图形预分析参数调用 MCP graph tools。
- 如果图形可用：得到风险等级 + 影响面 → 调整验证深度和优先级
- 如果图形不可用：跳过，继续步骤 3

### 3. 分析变更
```bash
git diff --name-only
```

### 4. 推导验证 + 证据
按技能中的映射表，根据变更类型匹配验证方法。

### 5. 执行输出
逐项 ✓（附证据）或 ✗（附报错）。
