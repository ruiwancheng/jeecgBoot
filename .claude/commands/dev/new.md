# /new

上下文膨胀时，打开一个全新的 pi 终端作为控制中心——带齐所有规则和状态，但没有历史噪音。

## 与 /delegate 的区别

| | /delegate | /new |
|------|:--:|:--:|
| 用途 | 派一个具体任务给工人 | 开一个干净的指挥部 |
| 终端 | 工人干完自动销毁 | 长期存活 |
| 卡片 | 卡片+工作流+任务 | 卡片+工作流+提示"这是新控制中心" |

## 流程

1. 运行 `/cleanup-context` 生成记忆卡片
2. 用 `orca terminal create` 创建新 pi 终端（不加任务——这不是工人）
3. 等待终端就绪（`orca terminal wait --for tui-idle`）
4. 用 `orca terminal send` 把记忆卡片 + 引导语发送到新终端
5. 提示用户："新控制中心已就绪，切换到那个终端即可开始工作"

## 发送内容格式

```markdown
你是 MES 项目的**新控制中心**——这个终端上下文干净，无历史噪音。

<记忆卡片——包含全部硬规则+工作流+禁止模式+项目状态>

---

你可以在这里：
- /brainstorm <需求> → /plan → 写代码
- /delegate <任务> → 派工人干活（上下文隔离）
- /verify → 编译+curl验证
- /quality-gate → 安全扫描

当前待部署提交：<N> 个。需要先部署还是继续攒？
```

## 使用示例

```
用户（在当前膨胀终端里）：/new
AI：📋 生成记忆卡片...
    🚀 创建新终端 term_xxx ✅
    📤 记忆卡片已发送 ✅
    
    ✅ 新控制中心已就绪：切换到 term_xxx 继续工作
    （当前终端可以关闭了）
```

## 降级

如果 Orca 不可用：退化为 `/cleanup-context` + 提示用户手动开终端粘贴记忆卡片。
