# /new

上下文膨胀时，打开一个全新的 pi 终端作为控制中心——带齐所有规则和状态，但没有历史噪音。

## 与 /delegate 的区别

| | /delegate | /new |
|------|:--:|:--:|
| 用途 | 派一个具体任务给工人 | 新终端自动续接当前工作 |
| 终端 | 工人干完自动销毁 | 长期存活 |
| 卡片 | 卡片+工作流+任务 | 卡片+恢复指令+自动继续 |

## 流程

1. 运行 `/cleanup-context` 生成记忆卡片（含会话上下文——最重要的增量信息）
2. 用 `orca terminal create --command "pi"` 创建新 pi 终端
3. 等待终端就绪（`orca terminal wait --for tui-idle`）
4. 用 `orca orchestration task-create` + `dispatch --inject` 把记忆卡片注入新终端
   - 和 `/delegate` 同样的 `--inject` 机制，卡片作为初始上下文注入，而非普通用户消息
5. 提示用户："新控制中心已就绪，切换到那个终端即可开始工作"

## 注入内容格式（task spec）

```json
{
  "title": "新控制中心",
  "description": "## 🤖 你是上一个会话的延续\n\n上下文已重置，无历史噪音。请先消化以下记忆卡片了解项目状态和当前会话上下文。\n\n**恢复指令：** 阅读卡片后，判断当前阶段直接继续工作：\n- brainstorm/plan 阶段 → 继续分析，不要跳到写代码\n- implement/verify 阶段 → 直接执行"下一步"中的动作\n- 如果卡片中没有明确的下一步 → 回复当前进度摘要，问用户要做什么\n\n<记忆卡片——含当前会话上下文+关键提醒+项目状态>\n\n---\n\n不要回复ready，不要问\"你要我做什么\"——直接继续。"
}
```

## 使用示例

```
用户（在当前膨胀终端里）：/new
AI：📋 生成记忆卡片... ✅
    🚀 创建新终端 term_xxx ✅
    📤 --inject 注入记忆卡片 ✅
    
    ✅ 新控制中心已就绪：切换到 term_xxx 继续工作
    （当前终端可以关闭了）
```

## 降级

如果 Orca 不可用：退化为 `/cleanup-context` + 提示用户手动开终端粘贴记忆卡片。
