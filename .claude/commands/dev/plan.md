# /plan <任务描述>

需求确认后制定实施计划。

**铁律：禁止"TODO"、"TBD"、"适当处理"。每步必须指定具体文件和操作。**

## 流程

### 1. 加载领域知识
使用 `plan` 技能获取：features.json 结构说明、标品概念、三种策略判断标准（纯新增/覆盖标品/加字段）。

### 2. 查模块
查 features.json 找到相关标品模块位置。

### 3. 模式锚定
读标品类似代码，引用具体文件和行号作为参照。

### 4. 判断策略
按技能中的三种策略标准判断。

### 4.5. 判断是否需要 orca-review（强制检查）
**除以下豁免外，一律触发 orca-review：**
- 免评：纯文本修改、注释修改、CSS/样式调整、列宽/标签修改
- 必评：Java/Vue/TS/SQL 任何非免评改动，无论文件数

→ **必评**：暂停，先派 Claude 做 orca-review，吸收评审意见后再输出最终计划
→ **免评**：跳过，直接继续步骤 5

**禁止跳过。** 如果 Orca 不可用，标注"降级：手工评审"并至少逐个对照依赖查证清单复查。

### 5. 输出计划
文件清单 + 任务步骤（含具体代码和验证命令）+ 范围外 + 风险。

### 6. 用户确认后执行
