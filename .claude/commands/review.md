# /review

审查未提交的变更。

## 流程

### 1. 加载领域知识
使用 `review` 技能获取：7 类审查定义（正确性/类型安全/模式合规/安全性/性能/完整性/可维护性）和 4 级严重度判定标准及动作。

### 2. 审查
对未提交变更按 7 类逐项检查，按 4 级严重度标记。

### 3. 修复落地
列出 MEDIUM 及以上问题，询问：
```
修复哪些？(1,2,3 / all / none)
```
确认的项自动修复。CRITICAL/HIGH 修复后重跑审查。
