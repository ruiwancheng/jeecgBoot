# /review

审查未提交的变更。

## 流程

### 1. 加载领域知识
使用 `review` 技能获取：7 类审查定义（正确性/类型安全/模式合规/安全性/性能/完整性/可维护性）、4 级严重度判定标准及动作、架构感知严重度调整参数。

### 2. 架构上下文获取（可选增强）
使用技能中的架构感知参数调用 MCP graph tools。
- 如果图形可用：获得变更函数的 hub/bridge/调用者数量 → 严重度自动加权
- 如果图形不可用：跳过，使用标准严重度逻辑

### 3. 审查
对未提交变更按 7 类逐项检查，按 4 级严重度标记。

### 4. 修复落地
列出 MEDIUM 及以上问题，询问：
```
修复哪些？(1,2,3 / all / none)
```
确认的项自动修复。CRITICAL/HIGH 修复后重跑审查。
